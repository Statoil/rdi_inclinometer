User guide

Software requirements

Install these on laptop before departure

  • Putty is a client that lets you connect to the SBC's via SSH.
  • Phidgets libraries for Windows are a set of tools that lets you interact with the sensors, including the important Compass calibration (installs by default at C:\Program Files\Phidgets\Examples)
  • Bonjour for Windows is useful for discovering SBC's more seamlessly.
  • Attach all sensors to laptop while being connected to the internet, to make sure drivers can be downloaded if necessary!

Routines

Start of Day

  • Before starting a new day of observations, Calibrate compass using the Compass calibration tool installed.
  • Update the configuration (height, and axis of phidget on binoculars) if needed.
  • Align with horizon

During day

  • Keep an eye on observations, to make sure gyro does not drift too much. Realign with horizon if necessary.

After day

  • Export data of the day, so that you are sure you have a backup source.

Other useful notes

  • Hostname SBC1: 192.168.2.11
  • Hostname SBC2: 192.168.2.12
  • Hostname SBC3: 192.168.2.13

With Putty installed, you will be able to connect to the SBC's through SSH. Host address for SBC1 is root@192.168.2.11 and for SBC2 it is root@192.168.2.12 password is mobydick

You can also connect to the PhidgetSBC configuration page for each SBC. SBC1 and SBC2. Username is admin, password is mobydick

If you need to connect to the router gateway interface, connect here. Username is admin, password is mobydick

If there are bugfixes done while on the trip, once you are online, you need to get the latest version of the code. Connect with putty to the SBC. Go to the directory /usr/userapps/rdi_inclinometer and type "git pull". If the code changes are complex, you might need to reboot the SBC. That can be done by connecting to the PhidgetSBC configuration page, and then going into "System" -> "Reboot".

The router has been preconfigured to give the SBC's static IP's, and the SBC's have been preconfigured to automatically start the application, and connect to the router's network. If this fails, everything you need to fix the problem can be found in the web interface for the PhidgetSBC, and the router web interface.
